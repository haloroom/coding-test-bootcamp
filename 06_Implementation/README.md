# 06_Implementation: 구현 및 시뮬레이션

## 핵심 개념 (Core Concept)
- **정의:** 문제에서 주어진 복잡한 조건, 동작, 제약 사항을 그대로 코드로 정확하게 옮기는 유형. 특정 알고리즘 지식보다는 논리적 사고와 꼼꼼함이 중요하다.
- **시뮬레이션 (Simulation):** 문제에서 제시된 과정이나 규칙을 시간의 흐름 또는 순서에 따라 그대로 따라가는 방식.

## 주요 로직 (Key Logic)
- **2차원 배열 조작:** 상하좌우 이동, 회전, 경계 검사 등 2차원 공간을 다루는 능력이 필수.
- **방향 벡터 (dx, dy):** `dx = [-1, 1, 0, 0]`, `dy = [0, 0, -1, 1]` 와 같이 방향을 배열로 정의하여 코드를 간결하게 만든다.
- **상태 관리:** 시간, 위치, 값 등 시뮬레이션 과정에서 변하는 모든 상태를 정확히 추적해야 한다.
- **예외 처리:** 문제의 모든 제약 조건과 코너 케이스를 빠짐없이 처리하는 것이 관건.

## 대표 예제 (Classic Examples)
- **상하좌우:** 주어진 이동 계획에 따라 최종 목적지 좌표를 계산.
- **왕실의 나이트:** 특정 위치에서 나이트가 이동할 수 있는 경우의 수.
- **로봇 청소기:** 주어진 맵과 규칙에 따라 로봇 청소기의 작동을 시뮬레이션.
- **주사위 굴리기:** 주사위의 상태를 관리하며 맵 위에서 이동하는 시뮬레이션.
- **기둥과 보 설치:** 주어진 조건에 맞게 건축물을 짓고 삭제하는 과정을 구현.
