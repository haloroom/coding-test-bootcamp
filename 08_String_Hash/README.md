# 08_String_Hash: 문자열 처리 및 해시

## 핵심 개념 (Core Concept)
- **문자열 알고리즘:** 텍스트 데이터를 효율적으로 검색, 분석, 조작하는 기법.
- **해시 (Hash):** 임의의 길이의 데이터를 고정된 길이의 데이터(해시 값)로 매핑하는 함수. 해시 테이블(해시 맵)은 이 원리를 이용해 $O(1)$의 평균 시간 복잡도로 데이터의 삽입, 삭제, 탐색을 가능하게 하는 자료구조.

## 주요 로직 (Key Logic)
- **해시 테이블 (Dictionary/Map):** 키-값 쌍으로 데이터를 저장하여 빠른 조회를 지원.
- **문자열 파싱/분석:** `split()`, `substring()` 등의 내장 함수와 정규 표현식(Regex)을 활용한 문자열 처리.
- **Trie (트라이):** 문자열 집합을 효율적으로 저장하고 검색하기 위한 트리 자료구조. 자동 완성, 접두사 검색 등에 유용.
- **라빈-카프 알고리즘 (Rabin-Karp):** 해싱을 이용하여 긴 텍스트에서 특정 패턴(문자열)을 빠르게 찾는 알고리즘.

## 대표 예제 (Classic Examples)
- **완주하지 못한 선수:** 참가자 명단과 완주자 명단을 해시 테이블로 비교하여 미완주자 찾기.
- **전화번호 목록:** 한 번호가 다른 번호의 접두어인지 판별. (해시 또는 Trie)
- **베스트앨범:** 장르별 총 재생 횟수와 노래별 재생 횟수를 해시 테이블로 관리하여 문제 해결.
- **문자열 검색:** 특정 키워드가 문서 내에 존재하는지 확인.
