# 03_Dynamic_Programming: 동적 계획법 (DP)

## 핵심 개념 (Core Concept)
- **정의:** 복잡한 문제를 여러 개의 작은 부분 문제로 나누어 풀고, 그 결과를 저장하여 동일한 계산의 반복을 피하는 알고리즘 기법.
- **조건:**
    1.  **중복되는 부분 문제 (Overlapping Subproblems):** 동일한 작은 문제가 여러 번 반복해서 나타난다.
    2.  **최적 부분 구조 (Optimal Substructure):** 문제의 최적해가 부분 문제의 최적해로 구성된다.

## 주요 로직 (Key Logic)
- **점화식:** 현재 문제와 부분 문제 간의 관계를 수식으로 표현. DP의 핵심.
- **메모이제이션 (Memoization):** 계산 결과를 배열이나 해시 테이블에 저장하는 것.
- **Top-down (하향식):** 큰 문제에서 시작하여 재귀적으로 작은 문제를 호출. (주로 재귀 + 메모이제이션)
- **Bottom-up (상향식):** 가장 작은 문제부터 시작하여 반복문을 통해 점차 큰 문제의 답을 채워나감. (주로 반복문 + DP 테이블)

## 대표 예제 (Classic Examples)
- **피보나치 수열:** `f(n) = f(n-1) + f(n-2)`
- **최장 증가 부분 수열 (LIS):** 주어진 수열에서 가장 긴 오름차순 부분 수열을 찾는다.
- **배낭 문제 (Knapsack Problem):** 한정된 용량의 배낭에 최대 가치를 담을 수 있는 물건 조합을 찾는다.
- **편집 거리 (Edit Distance):** 두 문자열이 같아지기 위해 필요한 최소 편집(삽입, 삭제, 교체) 횟수.
