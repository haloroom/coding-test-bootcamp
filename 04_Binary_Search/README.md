# 04_Binary_Search: 이분 탐색 (Binary Search)

## 핵심 개념 (Core Concept)
- **정의:** **정렬된** 배열이나 리스트에서 특정 값을 찾는 탐색 알고리즘. 시간 복잡도는 $O(\log N)$이다.
- **원리:**
    1. 탐색 범위의 시작(start), 끝(end), 중간(mid) 인덱스를 설정한다.
    2. 찾으려는 값과 중간 값을 비교한다.
    3. 값이 더 작으면 끝점을 `mid - 1`로, 값이 더 크면 시작점을 `mid + 1`로 갱신하여 탐색 범위를 절반으로 줄인다.
    4. 시작점이 끝점보다 커질 때까지 반복한다.

## 주요 로직 (Key Logic)
- **Start, End, Mid 포인터:** 세 개의 인덱스 변수를 이용해 탐색 범위를 관리. `mid = (start + end) // 2`
- **반복문 제어:** `while start <= end:` 루프를 통해 탐색을 계속하고, 종료 조건을 명확히 하는 것이 중요.
- **Parametric Search:** 최적화 문제를 결정 문제('yes' or 'no')로 바꾸어, 특정 조건을 만족하는 최댓값/최솟값을 이분 탐색으로 찾는 기법.

## 대표 예제 (Classic Examples)
- **정렬된 배열에서 특정 값 찾기:** 이분 탐색의 가장 기본적인 활용.
- **숫자 카드:** 특정 숫자가 배열에 존재하는지 확인.
- **공유기 설치:** 공유기 사이의 거리를 최대로 하는 문제. (Parametric Search)
- **예산:** 정해진 예산 내에서 최대로 배정할 수 있는 금액 찾기. (Parametric Search)
